unstable_hasFallbackAcl#

▸ unstable_hasFallbackAcl‹Resource›(resource: Resource): resource is Resource & unstable_WithFallbackAcl

Defined in acl.ts:194

Verify whether a fallback ACL was found for the given Resource.

A Resource fetched with its ACL (e.g. using unstable_fetchLitDatasetWithAcl) might have a fallback ACL attached, but we cannot be sure: the currently authenticated user (if any) might not have Control access to one of the fetched Resource's Containers.

This function verifies that the fallback ACL is accessible.

Type parameters:

▪ Resource: unstable_WithAcl

Parameters:

NameTypeDescription
resourceResourceA LitDataset that might have a fallback ACL attached.

Returns: resource is Resource & unstable_WithFallbackAcl

Whether dataset has a fallback ACL attached.

unstable_hasResourceAcl#

▸ unstable_hasResourceAcl‹Resource›(resource: Resource): resource is Resource & unstable_WithResourceAcl & unstable_WithAccessibleAcl

Defined in acl.ts:143

Verify whether an ACL was found for the given Resource.

A Resource fetched with its ACL (e.g. using unstable_fetchLitDatasetWithAcl) might have a resource ACL attached, but we cannot be sure: it might be that none exists for this specific Resource (in which case the fallback ACL applies), or the currently authenticated user (if any) might not have Control access to the fetched Resource.

This function verifies that the Resource's ACL is accessible.

Type parameters:

▪ Resource: unstableWithAcl & [WithResourceInfo](/lit-pod/docs/api/modules/_interfaces#withresourceinfo)

Parameters:

NameTypeDescription
resourceResourceA Resource that might have an ACL attached.

Returns: resource is Resource & unstable_WithResourceAcl & unstable_WithAccessibleAcl

Whether dataset has an ACL attached.
