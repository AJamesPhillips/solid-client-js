unstable_getAgentAccessModesAll#

▸ unstable_getAgentAccessModesAll(dataset: LitDataset & DatasetInfo & unstableAcl): *[unstable_AgentAccess](/lit-solid/docs/api/modules/_acl_agent#unstable_agentaccess) | null*

Defined in acl/agent.ts:62

Find out what Access Modes have been granted to specific Agents for a given LitDataset.

Keep in mind that this function will not tell you what access arbitrary Agents might have through other ACL rules, e.g. public or group-specific permissions.

Also, please note that this function is still experimental: its API can change in non-major releases.

Parameters:

NameTypeDescription
datasetLitDataset & DatasetInfo & unstable_AclThe LitDataset to which Agents may have been granted access.

Returns: unstable_AgentAccess | null

Which Access Modes have been granted to which Agents specifically for the given LitDataset, or null if it could not be determined (e.g. because the current user does not have Control Access to a given Resource or its Container).

unstable_getAgentDefaultAccessModesAll#

▸ unstable_getAgentDefaultAccessModesAll(aclDataset: unstable_AclDataset): unstable_AgentAccess

Defined in acl/agent.ts:166

Given an ACL LitDataset, find out which access modes it provides to specific Agents for the associated Container Resource's child Resources.

Keep in mind that this function will not tell you:

  • what access arbitrary Agents might have been given through other ACL rules, e.g. public or group-specific permissions.
  • what access arbitrary Agents have to the Container Resource itself (see unstable_getAgentResourceAccessModesAll for that).

Also, please note that this function is still experimental: its API can change in non-major releases.

Parameters:

NameTypeDescription
aclDatasetunstable_AclDatasetThe LitDataset that contains Access-Control List rules.

Returns: unstable_AgentAccess

Which Access Modes have been granted to which Agents specifically for the children of the Container associated with the given ACL LitDataset.
