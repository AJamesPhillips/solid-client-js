unstable_getPublicAccess#

▸ unstable_getPublicAccess(resourceInfo: unstableWithAcl & [WithResourceInfo](/solid-client-js/docs/api/modules/_interfaces#withresourceinfo)): unstable_Access | null

Defined in acl/class.ts:52

Find out what Access Modes have been granted to everyone for a given Resource.

Keep in mind that this function will not tell you what access specific Agents have through other ACL rules, e.g. agent- or group-specific permissions.

Also, please note that this function is still experimental: its API can change in non-major releases.

Parameters:

NameTypeDescription
resourceInfounstableWithAcl & [WithResourceInfo](/solid-client-js/docs/api/modules/_interfaces#withresourceinfo)Information about the Resource to which the given Agent may have been granted access.

Returns: unstable_Access | null

Which Access Modes have been granted to everyone for the given LitDataset, or null if it could not be determined (e.g. because the current user does not have Control Access to a given Resource or its Container).

unstable_getPublicDefaultAccess#

▸ unstable_getPublicDefaultAccess(aclDataset: unstable_AclDataset): unstable_Access

Defined in acl/class.ts:108

Given an ACL LitDataset, find out which access modes it provides to everyone for the associated Container Resource's child Resources.

Keep in mind that this function will not tell you:

  • what access specific Agents have through other ACL rules, e.g. agent- or group-specific permissions.
  • what access anyone has to the Container Resource itself (see unstable_getPublicResourceAccess for that).

Also, please note that this function is still experimental: its API can change in non-major releases.

Parameters:

NameTypeDescription
aclDatasetunstable_AclDatasetThe LitDataset that contains Access-Control List rules for a certain Container.

Returns: unstable_Access

Which Access Modes have been granted to everyone for the children of the Container associated with the given ACL LitDataset.

unstable_getPublicResourceAccess#

▸ unstable_getPublicResourceAccess(aclDataset: unstable_AclDataset): unstable_Access

Defined in acl/class.ts:80

Given an ACL LitDataset, find out which access modes it provides to everyone for its associated Resource.

Keep in mind that this function will not tell you:

  • what access specific Agents have through other ACL rules, e.g. agent- or group-specific permissions.
  • what access anyone has to child Resources, in case the associated Resource is a Container (see [[unstable_getDefaultResourceAccess]] for that).

Also, please note that this function is still experimental: its API can change in non-major releases.

Parameters:

NameTypeDescription
aclDatasetunstable_AclDatasetThe LitDataset that contains Access-Control List rules.

Returns: unstable_Access

Which Access Modes have been granted to everyone for the Resource the given ACL LitDataset is associated with.
